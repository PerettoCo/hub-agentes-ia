Próximos Passos · até quarta 16/07

A visão (o "cérebro digital" de cada cliente):

Imagine que cada cliente da V4 tem um centro de inteligência próprio composto por 4 camadas:

1. Banco de dados (Supabase + pgvector) — a fonte da verdade: clients, calls, transcripts, decisions, documents. Tudo indexado por similaridade semântica. O agente consulta direto — sem depender de arquivos .md locais.

2. Supabase + pgvector (banco de dados) — a memória de longo prazo: transcrições de calls indexadas por similaridade semântica, documentos referenciados, decisões registradas. O agente busca aqui sem precisar lembrar de nada — ele pergunta e o banco responde.

3. Google Drive (via MCP) — os arquivos reais do cliente: apresentações, planilhas, briefings, contratos. O agente acessa via Drive MCP sem baixar nada, só consulta, extrai e usa.

4. A mágica: Quando o account pergunta "qual o status do cliente?", o agente consulta a base, busca as últimas calls por similaridade, puxa o documento mais recente do Drive e monta um briefing completo em segundos — sem depender de arquivo local nenhum.

E mais: Com as tabelas de banco conectadas, o @flag-roi cruza verba de mídia com resultado de calls, o @evolucao-checkins gera relatório de progresso real, e cada cliente vira um mini-agente especializado que entende a fundo a conta consultando a base direto.

Caso real: Imagine que um designer entrega uma arte atrasada. O account vai lá no Drive, puxa a peça via MCP e pede avaliação. O mission control — que conhece o brandbook do cliente — sabe que azul é cor proibida para essa marca. Na hora que o account solicita a avaliação, o sistema alerta: "Essa peça usa azul, que viola a identidade visual do cliente conforme diretriz registrada no mission control." O problema é detectado antes de chegar ao cliente, o account não passa vergonha, e o designer recebe o feedback com a regra objetiva. Isso é o mission control funcionando como cérebro compartilhado entre funções.
