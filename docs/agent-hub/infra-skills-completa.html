🗄️ supabase-postgres-best-practices BANCO

Otimização de performance Postgres — 8 categorias priorizadas por impacto
O que resolve: Postgres mal otimizado é o gargalo #1 de qualquer stack de IA. Essa skill compila décadas de expertise do Supabase em 60+ regras práticas, cada uma com SQL errado → SQL certo → EXPLAIN → métricas de impacto. Toda query do SDR IA e dos dashboards segue essas regras.
# Antes (full table scan — 12s) SELECT * FROM orders WHERE status = 'pending' AND created_at > now() - interval '7 days'; # Depois (index scan — 40ms) CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_orders_status_date ON orders(status, created_at DESC); SELECT id, client_id, total FROM orders WHERE status = 'pending' AND created_at > now() - interval '7 days'; -- 12s → 40ms (300x mais rápido)

⏱️ temporal-python-testing WORKFLOWS

Testes de workflows Temporal com time-skipping, mocking e replay
O que resolve: Workflows que rodam por dias/meses são impossíveis de testar no tempo real. Temporal testing permite simular meses em segundos com time-skipping, mockar atividades para testar lógica pura, e validar determinismo com replay testing. Cobertura mínima de 80%.
# Teste com time-skipping: workflow de 30 dias em 2 segundos import pytest from temporalio.testing import WorkflowEnvironment @pytest.mark.asyncio async def test_onboarding_workflow(): env = await WorkflowEnvironment.start_time_skipping() async with Worker(env.client, task_queue="test", workflows=[OnboardingWorkflow], activities=[mock_activity]): result = await env.client.execute_workflow( OnboardingWorkflow.run, "cliente-X", id="test-onboarding", task_queue="test") assert result.status == "completed" await env.shutdown()

📦 hqq-quantization MODELOS

Half-Quadratic Quantization — 4/3/2-bit sem dados de calibração
O que resolve: Modelos grandes não cabem em GPU sem quantização. HQQ é a única técnica que permite 4/3/2-bit SEM precisar de dataset de calibração. Diferente de GPTQ e AWQ que exigem centenas de amostras, HQQ faz a quantização na hora. Essencial para rodar modelos locais no LiteLLM da VPS.

🧠 fine-tuning-with-trl MODELOS

Fine-tuning de LLMs com Reinforcement Learning — SFT, DPO, PPO
O que resolve: Quando os modelos base não são suficientes — precisamos fine-tunar com dados reais dos clientes (transcripts de check-in, campanhas que converteram, etc). TRL oferece SFT (instrução), DPO (preferência) e PPO/GRPO (recompensa). A stack completa open-source: TRL + HuggingFace + HQQ (quantização pós-treino) = sem depender de API paga.
