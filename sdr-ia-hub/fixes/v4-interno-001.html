V4 INTERNO · Ekyte 404

O workflow SDR IA tentava acessar boards e projetos no Ekyte com IDs inválidos ou desatualizados, recebendo Resource not found (404). Um nó Code de validação com mapa de boards válidos + IF router filtra requisições antes de enviar.

Explicação Simplificada

O que aconteceu?

O robô interno da V4 (V4 INTERNO) conversa com o Ekyte, que é uma ferramenta de gestão de projetos. Ele precisava buscar dados de boards (quadros) e projetos específicos.

O problema é que alguns IDs de boards que o robô estava usando eram antigos ou errados — como se você tivesse o endereço de uma loja que mudou de lugar. O Ekyte respondia "não encontrei isso aqui" (erro 404), e o robô não sabia lidar com essa resposta — ele simplesmente parava.

Isso acontecia em lote: quando um board dava 404, o processamento daquele lote inteiro ia pro espaço.

O que mudou?

Criamos uma "lista de endereços certos" (mapa de boards válidos) e colocamos um porteiro na porta: antes de perguntar algo ao Ekyte, o robô confere se aquele board existe na lista.

Se o board é válido, a requisição segue. Se não, o robô registra o erro e pula para o próximo — sem travar o lote inteiro.

É como ter uma agenda de telefones atualizada: antes de ligar, você confere se o número existe. Se não existe, não liga e já sabe que aquele contato está desatualizado.

Resultado esperado

Zero erros 404 no Ekyte. Boards inválidos são registrados e ignorados. O processamento em lote nunca mais trava por causa de um ID errado.

Detalhamento Técnico

Problema Original

Workflow: [V4 INTERNO] SDR IA — Gestão de Projetos

Nó com falha: HTTP Request (Ekyte API — GET /board/{id})

Erro: 404 — Resource not found

Causa raiz: O workflow iterava sobre uma lista de IDs de boards/projetos obtidos de uma fonte desatualizada (planilha estática). Alguns IDs tinham sido deletados ou renomeados no Ekyte, mas permaneciam na lista. O nó HTTP Request não tinha validação prévia nem tratamento de erro — ao receber 404, o nó lançava exceção e parava o fluxo inteiro do lote.

Impacto: Cada execução em lote processava N boards; se qualquer um retornasse 404, o lote inteiro falhava. Perda de dados dos boards válidos no mesmo lote.

Solução Aplicada

Inserir nó Code de validação + IF router antes da chamada HTTP:

const BOARD_MAP = {
  "projeto-alpha": "brd_a1b2c3d4",
  "projeto-beta": "brd_e5f6g7h8",
  "sdr-leads": "brd_i9j0k1l2",
  "v4-interno": "brd_m3n4o5p6",
  "campanhas-2026": "brd_q7r8s9t0"
};

const items = $input.all();
const validos = [];
const invalidos = [];

items.forEach(item => {
  const boardId = item.json.boardId || item.json.projetoId;
  if (BOARD_MAP[boardId]) {
    validos.push({
      json: { ...item.json, boardEkyteId: BOARD_MAP[boardId] }
    });
  } else {
    invalidos.push({
      json: {
        ...item.json,
        _erro: 'board_invalido',
        _mensagem: `Board ID "${boardId}" não encontrado no mapa de boards válidos`
      }
    });
  }
});

return [validos, invalidos];

Fluxo corrigido:

  • Trigger → Nó Code (valida + mapeia) → IF (rota válidos vs inválidos)
  • Válidos → HTTP Request Ekyte → processamento normal
  • Inválidos → Log + NoOp (registro para auditoria)

Arquivo de correção: v4-ekyte-fix-board.json (2 nós: Code + IF)

VersionId: correcao-v4-ekyte-v1

Antes vs Depois

Antes:

  • 404 quebrava o lote inteiro
  • Zero validação de entrada
  • IDs desatualizados na lista
  • Reprocessamento manual necessário

Depois:

  • Validação prévia contra BOARD_MAP
  • Router separa válidos de inválidos
  • IDs inválidos registrados em log
  • Zero interrupção no lote

Mitigação de Falhas

  • Board é renomeado no Ekyte → atualizar BOARD_MAP no nó Code (fonte única)
  • Board é deletado → BOARD_MAP desatualizado → item vai para inválidos, não quebra
  • Board ID vem em formato diferente → validação pode incluir regex de normalização
  • BOARD_MAP precisa ser atualizado com frequência → migrar para lookup table no banco

Plano de Rollback

Remover nó Code e IF, reconectar trigger direto ao HTTP Request. Erro 404 retorna, mas fluxo funcional anterior é restaurado.
