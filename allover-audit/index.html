Issues Críticos (P0)

P0-01. Site Hackeado — /about/ Comprometida com DAPURTOTO (Gambling ID)
Crítico
Problema: A página /about/ foi completamente comprometida com conteúdo de gambling indonésio. Title tag: "DAPURTOTO", H1: "DAPURTOTO", Idioma "id" (Indonésio), 1113 palavras sobre gambling, links externos para plazaitdotid.pages.dev, datamacau88.com, itdl.superampgoogle.com. Hreflang apontando para sites de gambling.
Impacto: GRAVÍSSIMO. Risco de desindexação total do site pelo Google. Penalidade manual. Perda de confiança do usuário. Risco de blacklist.
Ação Imediata: 1. Remover todo conteúdo injetado. 2. Resetar senhas (admin, FTP, DB, hosting). 3. Revisar plugins e temas em busca de backdoors. 4. Remover arquivos suspeitos (wp-content/uploads/). 5. Limpar .htaccess e wp-config.php. 6. Adicionar WAF (Cloudflare, Wordfence, Sucuri). 7. Escanear site completo com Wordfence. 8. Após limpeza, solicitar revisão no GSC.
P0-02. Idioma Incorreto — pt-BR em Todas as 156 Páginas
Crítico
Problema: Todas as 156 páginas HTML detectadas como pt-BR em vez de en. Uma empresa de roofing em Houston, TX servindo clientes americanos está sinalizando ao Google que seu conteúdo é em português brasileiro.
Impacto: ALTÍSSIMO. Google pode interpretar o site como conteúdo em português, suprimindo rankings para buscas em inglês ("roofers in Houston").
Correção: 1. WordPress → Settings → Site Language: mudar para "English (United States)". 2. Garantir lang="en-US" no HTML. 3. Verificar se plugins de tradução não forçam pt-BR. 4. Configurar wp-config.php com define('WPLANG', 'en_US').
P0-03. Zero Structured Data — Nenhuma Página Tem Schema Markup
Crítico
Problema: Nenhuma das 156 páginas contém structured data (JSON-LD, Microdata, RDFa). Zero rich snippets. Perda de LocalBusiness, Service, FAQ, Review, BreadcrumbList, Article, Product schema.
Impacto: Perda massiva de visibilidade em SERPs. Concorrentes com schema LocalBusiness aparecem com estrelas, horário, endereço e botão de chamada — este site aparece como link simples. CTR estimado 20-30% inferior.
Correção: 1. LocalBusiness schema global (JSON-LD). 2. BreadcrumbList schema. 3. Article schema em blog posts. 4. FAQ schema em páginas informacionais. 5. Service schema em páginas de serviço. 6. Verificar com Google Rich Results Test. Configurar via Yoast SEO ou RankMath.
P0-04. Meta Descriptions Duplicadas — Elementor Injetando 2 Descrições
Crítico
Problema: Cada página tem DUAS meta descriptions. A primeira (300-400 chars) gerada automaticamente pelo Elementor, contendo texto repetido do corpo da página. A segunda (120-155 chars) é a correta. Google usa a primeira — extremamente longa e de baixa qualidade.
Impacto: Google ignora ambas ou gera description automática. Perda de controle sobre texto exibido no SERP.
Correção: 1. Desabilitar geração automática no Elementor → Settings → SEO. 2. Ajustar template do Elementor para não sobrescrever Yoast/RankMath. 3. Manter meta descriptions únicas de 120-155 chars com CTA e keyword primária.

Issues Técnicos

Homepage com 5.326ms de Resposta — Muito Lenta
Crítico
Problema: Tempo de resposta do servidor para a homepage é de 5.326 segundos. Vários blog posts também excedem 5s. Tamanho da homepage: 254KB só de HTML. O recomendado para TTFB é abaixo de 200ms.
Impacto: Core Web Vitals reprovados. Google prioriza sites rápidos. Usuários abandonam após 3s.
Correção: 1. Implementar cache de página (WP Rocket, W3 Total Cache, Flying Press). 2. Habilitar compressão Gzip/Brotli. 3. Otimizar banco de dados WordPress. 4. Implementar CDN (Cloudflare — verificar configuração). 5. Reduzir número de plugins (Elementor Pro + Essential Blocks + LeadConnector + GHL). 6. Considerar upgrade de hospedagem. 7. Otimizar imagens.
12 Imagens Acima de 300KB — Alguns Monstros de 1.7MB
Alto
Problema: 12 imagens com tamanhos excessivos: commercial-roofing-options-4.jpg (1.757.537 bytes / 1.7MB!), how-long-metal-roofs-last.webp (1.385.622 bytes), commercial-roofing-contractor.webp (1.177.486 bytes).
Impacto: Impacto direto em LCP e tempo de carregamento.
Correção: 1. Usar ShortPixel, Imagify ou Smush para compressão em lote. 2. Redimensionar para max 1920px. 3. Converter para WebP com fallback. 4. Implementar lazy loading. Alvo: <100KB para imagens de conteúdo.
Imagens SEM Dimensões (width/height) — Risco de CLS
Alto
Problema: Nenhuma das 149 imagens auditadas tem atributos width e height definidos. Isso causa Cumulative Layout Shift (CLS) — o layout "pula" conforme as imagens carregam.
Correção: Adicionar width e height em todas as tags <img>. No Elementor, as imagens devem ter dimensões definidas no widget. Verificar se tema ou plugin está removendo esses atributos.
