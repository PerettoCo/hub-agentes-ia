Arquitetura

O Ekyte é o centro de tudo. Os agentes OpenCode usam 65 tools MCP para ler e escrever no Ekyte — tarefas, projetos, tickets, horas, quadros. O Google Drive fornece documentos como contexto. O LiteLLM roteia chamadas para os modelos certos com governança por squad. O n8n automatiza workflows em background.

🔗 Como os componentes se conectam:
OpenCode → Ekyte: via MCP (65 tools) — cria tasks, tickets, projetos
OpenCode → Drive: via MCP (8 tools) — lê briefings, salva relatórios
OpenCode → LiteLLM: via API — roteia prompts para o modelo certo
n8n ↔ Ekyte: via webhook — ouve e escreve eventos
Ekyte e Drive não se conectam diretamente — o agente ou o n8n é a ponte

Fluxos

Como os agentes e o Ekyte trabalham juntos nos fluxos reais de cada squad.

Pós-Call: Processar Check-in

  1. Usuário cola o transcript da call no agente Ekyte
  2. @account-orchestrator extrai combinados e prazos
  3. Para cada combinado: create_task + create_task_comment
  4. Se detectou risco: create_ticket (flag)
  5. Drive MCP: create_file com resumo da call
  6. Retorna: "3 tarefas criadas, 1 flag aberta, resumo salvo no Drive"

Pré-Call: Preparar Check-in

  1. @account-checkin-roleplay: "Me prepara pro check-in com cliente X"
  2. list_tasks → tasks pendentes do cliente
  3. search_files → pauta anterior no Drive
  4. list_tickets → flags e pendências
  5. Gera resumo: pauta, tasks, histórico, perguntas sugeridas

Criação de Conteúdo com Aprovação

  1. Account cria task no Ekyte: "Produzir LP Campanha Verão" create_task
  2. @copy-content lê briefing: search_files + read_file_content
  3. Gera rascunho via gpt-4o-mini
  4. create_file → salva rascunho v1 no Drive
  5. create_task_comment + update_task_phase → avança para revisão
  6. Ciclo de revisão até aprovação, então update_task como concluída

Análise de Tráfego com Ação

  1. GT cria task: "Analisar ROAS cliente X" create_task
  2. @media-buyer puxa dados reais via V4mos API
  3. list_tasks → histórico de campanhas do cliente
  4. get_detailed_project → orçamento disponível
  5. Se ROAS abaixo da meta: @flag-roi → create_ticket
  6. Drive: create_file com relatório
  7. Ekyte: create_task_comment + update_task_phase

Comitê de P&EG Automatizado

  1. n8n (cron domingo 20h): list_tasks + list_tickets + list_time_trackings por squad n8n
  2. n8n consolida em JSON e salva no Drive
  3. @executor-comite "Gera briefing do comitê"
  4. Lê dados do Drive read_file_content
  5. Convoca @analista-dados e @revisor
  6. Drive: salva briefing .md + .html
  7. n8n envia briefing por email

Estratégia de Otimização

  • DeepSeek V4 Flash como padrão — gratuito, usado em 70% das operações
  • Gemini 2.5 Flash gratuito — para análise de documentos grandes
  • GPT-4o Mini — tarefas em português que exigem mais precisão
  • Claude Sonnet 4 — apenas Design e Coordenação (tarefas estratégicas)
  • Budget por squad no LiteLLM para evitar estouro de custos

Verificação Final

Ao final, o membro deve conseguir:

  • Listar tasks do squad no Ekyte
  • Buscar documentos no Drive via agente
  • Rodar o agente principal do squad
  • Criar tarefas no Ekyte via agente
  • Salvar arquivos no Drive via agente
