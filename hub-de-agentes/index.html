O Ecossistema: 101 Skills · 36 Agentes · 8 Times

O Hub de Agentes V4 transforma o Builders Hub em uma força de trabalho orquestrada. Cada skill é um instrumento, cada agente é um músico, e o Opensquad é o maestro.

📊 Em números: 101 skills em `.agents/skills/` · 36 agentes em `.opencode/agents/` · 8 times orquestrados · 10+ formações de time · 6 modelos de IA gratuitos · 4 squads operacionais · Opensquad como backbone
⚙️ Funções Nativas do Builders Hub: Antes das skills especializadas, existem as funções base do repositório — o "sistema operacional". /onboarding configura o ambiente, /contexto atualiza o conhecimento do agente sobre um cliente, /sabatina stress-test de planos, /sync-hub puxa skills novas do time. Use /contexto sempre que terminar uma seção de trabalho com um cliente — o agente precisa do contexto atualizado. Veja todas na seção de Base.

Diagrama da Arquitetura

Opensquad — O Backbone de Orquestração

O Opensquad é o gestor dos squads. Ele cria, configura, executa pipelines e mantém memória entre sessões. Cada squad Opensquad tem seu pipeline (squad.yaml), seus membros (squad-party.csv) e sua memória (memories.md).

Capacidades do Opensquad

CapacidadeComo Funciona
OnboardingColeta dados da empresa, pesquisa website, gera company.md com perfil completo
Criação de SquadsCria squads com pipeline (squad.yaml), agentes (squad-party.csv) e memória persistente
Execução de PipelineLê squad.yaml, carrega agentes do CSV, executa passo a passo com checkpoints
MemóriaRegistra aprendizados em memories.md após cada execução — conhecimento acumula
SkillsCatálogo de skills instaláveis — browse, install, uninstall, create
15+ Comandos/opensquad, create, run, list, edit, delete, skills, install, uninstall, edit-company, settings, reset
Multi-idiomaOutput no idioma do usuário, arquivos internos em inglês

Como Opensquad e Skills se Conectam

Conceito OpensquadNo Hub V4
squad.yaml (pipeline)Fluxo de cada squad: Account → Pré-call → Call → Pós-call; GT → Análise → Otimização → Reporte
squad-party.csv (membros)Agentes do squad: @account-orchestrator, @media-buyer, @criacao-design...
company.md (contexto)CLAUDE.md + AGENTS.md da V4 — missão, valores, metodologias V4
memories.md (aprendizado)Ekyte como memória viva: tasks, comentários, histórico de execução
SkillsInstruções especializadas em .agents/skills/ — o conhecimento de cada agente
⚡ A diferença: Opensquad é o gestor — cria squads, executa pipelines, mantém memória. As skills são o conhecimento — o "como fazer" de cada tarefa. Opensquad sem skills é um maestro sem orquestra. Skills sem Opensquad são instrumentos sem músico.

Funções Nativas do Builders Hub — Base (13)

⚙️ Essas são as funções nativas do repositório. Elas vêm com o Builders Hub original da matriz e são o "sistema operacional" de tudo. Antes de usar qualquer skill especializada, conheça estas:

/onboarding → roda na primeira vez que clonar o repo
/contexto → roda sempre que começar/terminar trabalho num cliente
/sabatina → roda antes de decisões importantes
/sync-hub → roda para puxar skills novas do time

Dica: terminou de usar o agente com um cliente? Roda /contexto. O agente precisa do contexto atualizado para funcionar bem.

O Que Esperar do Refinamento

Cada skill em .agents/skills/{nome}/SKILL.md será ajustada nos seguintes aspectos:

  1. Descrição — está clara para o OpenCode saber quando invocar?
  2. Triggers — adicionar termos que o time usa no dia a dia
  3. Workflow — o passo-a-passo reflete a realidade da área?
  4. Input esperado — o que o usuário precisa fornecer?
  5. Output gerado — formato correto?
  6. Integrações — quais MCPs/APIs precisa acessar?
⚠️ Estado atual: As skills estão na versão rascunho v1 — construídas a partir de documentos, não de validação com os times. O refinamento pós-sessões é obrigatório antes de considerar qualquer skill como "pronta". Consulte docs/agents-hub/TASKS.md para o plano completo.
